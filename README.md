# k1-surf — деплой и оптимизация

Репозиторий для контролируемых правок сайта **k1.surf** (Bitrix, шаблон concept_phoenix_s1)
с автодеплоем на хостинг Timeweb по FTP.

## Как устроено
- Папка `site/` зеркалит структуру сайта от корня `public_html`.
  Всё, что лежит в `site/`, при push в `main` автоматически заливается на сервер.
- `.github/workflows/deploy.yml` — сам автодеплой (GitHub Actions + FTP).

## Что сейчас в репозитории
- `site/.htaccess` — оптимизированный конфиг (сжатие, кеш браузера, WebP, заголовки).

## Первый запуск
1. Создать 3 секрета: `FTP_SERVER`, `FTP_USERNAME`, `FTP_PASSWORD`.
2. Проверить путь `server-dir` в `deploy.yml`.
3. `git push` (или правка через веб GitHub) → вкладка **Actions** покажет лог деплоя.

## Дальше
В `site/bitrix/templates/concept_phoenix_s1/` добавляются правки шаблона
(размеры картинок для CLS, preload баннера для LCP, отложенная Метрика) —
после push они едут на сайт автоматически.

> ⚠️ Ядро Bitrix (`/bitrix/`), `/upload/` и пароли БД в репозиторий не кладём.
